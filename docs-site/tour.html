Values

Penelope has seven runtime value tags: int, bool, str, unit, closure, list, dict.

42           // int
true         // bool
"hello"      // str
list_new(1, 2, 3)              // list
dict_set(dict_new(), "k", 1)  // dict
fn(x) { x + 1 }                // closure

Let bindings

let x = 10;
let y = x + 5;
print(to_str(y));  // 15

Functions are values

let add = fn(a, b) { a + b };
print(to_str(add(2, 3)));  // 5

// Closures capture lexical scope
let mk_counter = fn(start) {
  fn(n) { start + n }
};
let add10 = mk_counter(10);
print(to_str(add10(5)));  // 15

Conditionals

let abs = fn(n) {
  if (n < 0) { 0 - n } else { n }
};
print(to_str(abs(0 - 7)));  // 7

The headline feature: pause

pause is a runtime primitive. When evaluated, it serializes the entire program state and exits the process. A later pen resume picks up exactly where it left off.

// pause.pen
let x = 10;
let y = pause;
print(to_str(x + y));
$ bin/penelope run pause.pen
paused at ip 2 → pause.penz

# process exits — could be hours, days, years later

$ bin/penelope resume pause.penz
10
# x survived; bare pause returns unit; the print fires on resume

Effects: I/O that survives pause

All side effects (print, file I/O, network, time, randomness) flow through an effect log. On resume, completed effects are replayed from the log — they don't re-execute. This guarantees idempotency across pause boundaries.

let response = net_fetch("https://example.test/decision");
let ok = wait_for("approval");
write_file("/tmp/audit.log", response);
print("audit complete");

The 8 effects: print, net_fetch, now, random_int, read_file, write_file, wait_until, wait_for.

wait_for: external value injection

When a program hits wait_for(name), the runtime pauses. A later pen resume --event name=value injects the value and resumes.

$ pen run hitl.pen          # pauses on wait_for("approval")
$ pen resume hitl.penz --event approval=true

Forking: two futures from one snapshot

$ pen fork pause.penz pause-copy.penz
# Now you have two snapshots. Resume each independently — they diverge.

String interpolation

Strings may embed expressions with ${...}. Each ${expr} is desugared to to_str(expr) + concat.

let name = "Penelope";
let age = 42;
print("hello ${name}, you are ${age + 1}");
// → hello Penelope, you are 43

Escape a literal $ with \$.

Pattern matching

match expr { pattern => body, ... }. Patterns supported (MVP): int / bool / string literals, wildcard _, variable binding x.

let describe = fn(n) {
  match n {
    0 => "zero",
    1 => "one",
    x => "many: ${x}",
  }
};
print(describe(42));   // → many: 42

Arms are checked top-to-bottom. A variable pattern (lowercase identifier) matches anything and binds; _ matches anything and discards. No exhaustiveness checking — a non-matching scrutinee with no wildcard returns unit.

Pattern variants

Or-patterns: 1 | 2 | 3 => "small" matches any alternative.

Guards: n if n > 100 => "huge" — arm fires only if the guard expression is true (after pattern bindings).

List patterns: [] => ... matches empty list; [h, ...t] => ... binds head and rest; [a, b] => ... requires exactly two elements.

let head = fn(xs) {
  match xs {
    [] => "empty",
    [h, ...t] => "head=${h}, len(t)=${list_len(t)}",
  }
};

Dict patterns: {name: n} => ... binds n to the dict's name field; missing key falls through.

let lookup = fn(d) {
  match d {
    {name: n} => "name=${n}",
    _ => "no name",
  }
};

Syntax niceties

Negative literals (-5), hex (0xff), binary (0b1010), underscore separators (1_000_000), block comments (/* ... */), and the unit literal ().

Modules

import "./path.pen"; expands the imported file's top-level bindings into the current scope. Paths are resolved relative to the importing file. Re-imports are idempotent; cycles are safe.

// math.pen
let square = fn(n) { n * n };
// main.pen
import "./math.pen";
print(to_str(square(7)));    // 49

The loader expands imports before tokenization. Source positions in errors refer to the inlined source — preserving original file:line is a future improvement.

Optimizations

Compile with -O2 to enable all 5 passes (constant folding, dead-code elimination, inline caches, function inlining, peephole):

$ pen build -O2 fib.pen
wrote fib.penc (14 opcodes, 2 constants, -O2)
$ pen disasm fib.penc
